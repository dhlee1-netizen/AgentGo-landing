## CORE (플랫폼 구성)

**Label**: 플랫폼 구성  
**H2**: 개발부터 거버넌스까지 하나의 플랫폼  
**Desc**: 에이전트 전체 라이프사이클을 단일 환경에서 관리합니다.

| Tag | 모듈 | 설명 | Pills |
|-----|------|------|-------|
| 인프라 | Platform | 온프레 · 클라우드 · 폐쇄망. 벤더 Lock-in 없이 배포. | K8s, 온프레, 폐쇄망 |
| 보안 | Guard | 권한·가드레일·감사 로그. 업무 규정을 자동 반영. | 접근 제어, 감사 로그, 데이터 주권 |
| 운영 | Management | 상태·사용량·비용을 실시간 대시보드로 통합. | 모니터링, 비용 분석, 상태 관리 |
| 마켓플레이스 | Marketplace | 검증된 에이전트를 즉시 배포. 개발 없이 자동화 시작. | 즉시 배포, 원클릭 설치 |
| 개발 | Coder | MCP 기반 에이전트 개발. 파편화된 AI를 하나로 통합. | MCP, SDK, API 연동 |
| 자동화 | Flow | 노코드 워크플로 설계·실행. 대화로 업무 시작. | 노코드, 스케줄링 |

- **기능**: RAG 문서 Q&A · Management 포함 · 에이전트 모니터링

- **부제**: 업무 자동화

## PROCESS (도입 프로세스)

**Label**: 도입 프로세스  
**Desc**: PoC를 통해 빠르게 검증하고, 전사 운영 체계까지 단계적으로 확장합니다.

| 단계 | 기간 | 설명 |
|------|------|------|
| 1. 적용 시나리오 정의 | 1~2주 | 대상 업무 선정 및 PoC 범위 협의 |
| 2. PoC 검증 | 1~2개월 | Use Case 검증 및 적용 가능성 확인 |
| 3. 전사 확산 및 운영 고도화 | 3개월~ | 파일럿 적용 → 운영 확장 → 전사 적용 |

## FAQ

**Label**: FAQ  
**H2**: 자주 묻는 질문

| 질문 | 답변 |
|------|------|
| 데이터가 외부로 나가지 않나요? | 온프레·폐쇄망 배포를 지원합니다. Guard가 접근을 암호화하고 감사 로그를 기록합니다. |
